# 用 Markdown 制作网站，只需几分钟

分享者：程序员闪电侠

## 你是如何判断这是一个值得投入的商机？

身边有一个不懂技术的小伙伴，自学了 HTML, CSS, JavaScript，自己买了域名，服务器，学着自己部署。做了一个个人业务介绍网站。
我正好在帮国外 HUGO 社区打造一个 Headless CMS 开源项目。这个全靠热情，因为我喜欢优秀开源项目源码，Hugo 代码库有十几年历史了。
我就想能不能做一个服务，大家只需要会写 Markdown 就行，不需要懂技术，就能快速搭建一个网站。
所以我做了 MDFriday - https://mdfriday.com

## 在你进入之前，这个市场的真实需求是什么？你如何验证它？

目前做网站的服务有 WordPress, Wix, Squarespace 等等。 
但是这些服务都比较重，功能复杂，价格也比较高。
哪怕只是一个部署服务，Obsidian 官网报价都要 8 美元每月起步。

最最最大的问题是，传统的这些建站平台，不是面向语言设计的，是面向介面设计的，都是需要在页面上操作。
而我想的是，在 大语言模型 时代，肯定能做到用自然语言交流就能做网站， 而 markdown 语言正好是一个很好的中间媒介。

因为这个市场一直存在，我只是提供了一个新的渠道，所以我不用再去验证这个市场到底存不存在。

## 第一个付费用户来自哪里？你做了什么让他们愿意付费？

我的服务还没有接入 支付 功能。
这是我接下来要做的最重要的事情。
而且我现在提供的都是免费主题。

哪怕是这样，国外已经有人打赏了我，虽然不多。
因为我一直在 Reddit 上宣传这个项目。

国内也有小伙伴支付了服务费，虽然也不多。

## 你做这款产品的驱动因素是什么：痛点、自用需求、技能优势，还是市场洞察？

我不想打一辈子工。

我想找到后面几十年，能一直坚持做的事情。
而不是给自己一个借口，等我退休了，我要去......

---

总结：如何用 Markdown 做网站，并找到值得坚持的产品方向

- 我看到很多人想做网站，但被技术门槛、复杂工具和高成本劝退，于是判断“用 Markdown 建站”是一个长期存在的真实需求。
- 传统建站工具偏向界面操作，而在大语言模型时代，内容驱动的建站一定会成为趋势，Markdown 是天然的桥梁。
- 我不需要验证市场是否存在，只需要证明我能提供更简单的路径。Reddit 和国内用户的早期打赏与付款，是最直接的验证。
- 我的驱动力来自长期的兴趣、技术积累、以及“想做一件能坚持几十年”的渴望，而不是打一辈子工。
